## 遇到的最大一个坑

**Windows 终端编码问题**：Windows 控制台默认使用 GBK 编码，导致打印中文和 emoji 时出现乱码。

解决方案：
1. 移除 emoji 符号（如 🚀、☑、☐）
2. 使用 ASCII 替代符号（如 [x]、[ ]）
3. 简化输出，避免复杂字符

**另一个坑**：DeepSeek API 返回的内容有时包含特殊字符（如 `\udca8`），这些是 UTF-8 的无效代理字符，会导致编码错误。但工具函数本身工作正常，只是显示有问题。
